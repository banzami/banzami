Create Payment Request

Pull-payment model: merchant creates a request, consumer pays it. The created ID auto-fills the Pay form below. A request can only be paid once.

Pay Payment Request

Paying a request creates a transfer for the exact requested amount. Attempting to pay an already-paid request returns 422.

Generate QR

Generates a BANZAMI-SBX:{base64} payload. In production, encode this string into a real QR image using any QR library. See contracts/qr/ for the production format.

Pay QR

QR codes are single-use. A second payment attempt returns 422. The QR ID auto-fills from the Generate form.

Wallet Ledger

Double-entry bookkeeping: every transfer creates exactly two entries — a DEBIT on the sender's account and a CREDIT on the receiver's. The balance shown is computed as SUM(CREDIT) − SUM(DEBIT) and is never stored separately. Try entering both sides of a transfer to see the symmetry.

Simulate Settlement

Settlement converts wallet receipts into bank disbursements. The sandbox applies a simulated 1% platform fee (minimum 1.00 AOA) and always completes instantly. In production, this triggers a real bank transfer via the settlement provider.

Settlement Batches

Each batch settles all unsettled inbound transfers for a wallet. A transfer can only be settled once.
